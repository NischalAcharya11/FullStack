HTML — Structure and Semantics

HTML (HyperText Markup Language) provides the structural foundation of web pages. Good HTML is semantic, accessible, and organized. It uses elements that describe meaning (e.g., <header>, <nav>, <main>, <article>). Semantic HTML not only improves accessibility for screen readers but also helps search engines understand content better, improving SEO. Well-structured HTML makes a page easier to style with CSS and more predictable when adding interactivity with JavaScript.

Think of HTML as the "skeleton" of a webpage. CSS is like its "skin and clothing" (appearance), and JavaScript gives it "movement and behavior" (functionality).

Core Principles
  • Use headings (<h1>–<h6>) in logical order for clear document hierarchy.
  • Wrap the main content in <main> and navigation in <nav> for clarity.
  • Prefer semantic containers (<section>, <article>, <aside>, <footer>) over generic <div> tags.
  • Add alt text to images so screen readers can describe visuals to users with disabilities.
  • Always label form inputs with <label> to improve accessibility and usability.
  • Keep markup clean and minimal; unnecessary nesting makes pages harder to maintain.
  • Ensure mobile-friendliness by using the correct viewport meta tag and responsive design practices.
Example Markup
<article class="post">
  <header>
    <h2>Post Title</h2>
    <p>Published <time datetime="2025-08-25">Aug 25, 2025</time></p>
  </header>
  <p>Clear, semantic HTML improves accessibility and SEO.</p>
  <footer>© Author</footer>
</article>

In this example, <article> groups self-contained content, <header> contains introductory information, and <time> provides machine-readable publication dates. This makes the document meaningful both to users and to software (browsers, crawlers, screen readers).

Forms and Accessibility

Forms are essential for user interaction — from signing up and logging in to searching or submitting feedback. Accessible forms ensure that all users, including those using assistive technologies, can interact with them effectively. Best practices include:

  • Always use <label> elements tied to inputs via the for attribute.
  • Provide clear instructions and feedback messages.
  • Use ARIA attributes (e.g., aria-describedby) when extra explanation is needed.
  • Mark required fields with the required attribute.
  • Ensure form controls can be reached and submitted using only a keyboard.
<form>
  <label for="email">Email</label>
  <input id="email" type="email" required aria-describedby="help" />
  <div id="help">We'll never share your email.</div>
  <button type="submit">Submit</button>
</form>

This form example demonstrates semantic HTML combined with accessibility. The input is linked to a label for clarity, the required attribute enforces validation, and aria-describedby ensures that assistive technologies can announce helpful instructions.
